# Slideスキル リファレンス一覧

このディレクトリには、Marp形式のプレゼンテーションスライドを生成する際に参照すべきガイドラインとリファレンスが含まれています。

## リファレンス一覧

### 1. visual-hierarchy.md
**用途**: ビジュアル階層とデザインガイドライン

スライドのデザイン統一性を保証するためのガイドラインです。フォントサイズ、カラースキーム、スペーシングなどの視覚的な階層を定義しています。

**含まれる内容**:
- フォントサイズ階層（h1、h2、h3、body）
- カラースキーム定義（プライマリ、セカンダリ、アクセント）
- スペーシングルール（マージン、パディング）
- コントラストとアクセシビリティ
- レイアウトパターン

**いつ参照するか**:
- スライドのスタイル定義を作成する際
- テンプレートをカスタマイズする際
- デザインの一貫性を確認する際

---

### 2. emoji-usage.md
**用途**: 絵文字使用ガイドライン

スライドに適切な絵文字を使用するためのガイドです。カテゴリ別に整理された絵文字リストと、使用のベストプラクティスを提供します。

**含まれる内容**:
- カテゴリ別絵文字一覧（10カテゴリ以上）
- 使用シーン別推奨
- 1スライド当たりの推奨個数
- アンチパターン（避けるべき使用例）
- ベストプラクティス

**いつ参照するか**:
- スライドに絵文字を追加する際
- 視覚的な魅力を向上させたい際
- カテゴリに適した絵文字を選びたい際

---

### 3. marp-syntax.md
**用途**: Marp構文リファレンス

Marp Markdownの正確な構文とベストプラクティスを記載したリファレンスです。フロントマター、スライド区切り、レイアウト指定などの基本構文を網羅しています。

**含まれる内容**:
- フロントマター設定（theme、paginate、header、footer）
- スライド区切り（`---`）
- レイアウト指定（`<!-- _class: lead -->`など）
- 画像挿入の方法
- リストとコードブロックの構文
- カスタムスタイルの適用方法

**いつ参照するか**:
- Marpマークダウンを生成する際
- レイアウトを指定したい際
- カスタムスタイルを適用する際

---

## リファレンスの使い方

### ステップ1: 要件に応じたリファレンスの選択

目的に応じて、適切なリファレンスを選択してください：

```
デザインの一貫性を確保したい
  → visual-hierarchy.md

適切な絵文字を選びたい
  → emoji-usage.md

Marp構文を確認したい
  → marp-syntax.md
```

### ステップ2: ガイドラインの適用

各リファレンスに記載されたガイドラインをスライド生成時に適用します。

**例: visual-hierarchy.mdの適用**

```yaml
style: |
  h1 {
    font-size: 48px;  # visual-hierarchy.mdに準拠
    color: #1a202c;
  }
  h2 {
    font-size: 36px;  # visual-hierarchy.mdに準拠
    color: #2d3748;
  }
```

**例: emoji-usage.mdの適用**

```markdown
# TypeScript入門 📚  # 学習カテゴリから選択

学習目標を確認しましょう 🎯  # 目標カテゴリから選択
```

**例: marp-syntax.mdの適用**

```markdown
---
marp: true
theme: default
paginate: true
---

<!-- _class: lead -->
# タイトルスライド

---

# 通常のスライド
```

### ステップ3: 整合性チェック

生成したスライドが、すべてのリファレンスに準拠しているか確認します。

**チェックリスト**:
- [ ] フォントサイズが階層に従っている（visual-hierarchy.md）
- [ ] カラースキームが統一されている（visual-hierarchy.md）
- [ ] 絵文字が適切に使用されている（1スライド1-2個まで、emoji-usage.md）
- [ ] Marp構文が正しい（marp-syntax.md）
- [ ] スライド区切りが正しく配置されている（marp-syntax.md）

## リファレンス間の関係

各リファレンスは独立していますが、相互に補完し合う関係にあります：

```
visual-hierarchy.md
    ↓ (デザインの基盤)
emoji-usage.md
    ↓ (視覚的な装飾)
marp-syntax.md
    ↓ (技術的な実装)
完成したスライド
```

### 優先順位

1. **marp-syntax.md**: 構文が正しくないと、スライドが正しく表示されない
2. **visual-hierarchy.md**: デザインの一貫性は、スライドの品質に直結
3. **emoji-usage.md**: 視覚的な魅力を向上させる補助的な要素

## ベストプラクティス

### 1. リファレンスを事前に読む

スライドを生成する前に、各リファレンスを一読することをお勧めします。特にvisual-hierarchy.mdとmarp-syntax.mdは重要です。

### 2. テンプレートとの併用

テンプレート（templates/）とリファレンス（references/）は併用することで、最大の効果を発揮します。

- テンプレートが「構造」を提供
- リファレンスが「品質」を保証

### 3. カスタマイズ時も準拠

テンプレートをカスタマイズする際も、リファレンスのガイドラインを守ることで、デザインの一貫性が保たれます。

### 4. 段階的な適用

すべてのガイドラインを一度に適用しようとせず、段階的に適用していくことをお勧めします：

1. **Phase 1**: marp-syntax.mdに従って基本構造を作成
2. **Phase 2**: visual-hierarchy.mdに従ってスタイルを適用
3. **Phase 3**: emoji-usage.mdを参照して絵文字を追加

## 参考情報

### Marp公式ドキュメント

リファレンスは、Marp公式ドキュメントに基づいています：

- [Marp公式サイト](https://marp.app/)
- [Marpit Markdown](https://marpit.marp.app/markdown)

### デザインの原則

visual-hierarchy.mdは、以下のデザイン原則に基づいています：

- **階層**: 情報の重要度を視覚的に表現
- **コントラスト**: 読みやすさとアクセシビリティ
- **一貫性**: 予測可能で統一されたデザイン
- **スペーシング**: 情報の整理と視覚的な快適さ

### 絵文字の選択基準

emoji-usage.mdは、以下の基準で絵文字を選定しています：

- **ユニバーサルデザイン**: 文化や背景に関わらず理解しやすい
- **プロフェッショナル**: ビジネス環境でも使用可能
- **視認性**: スライド上で見やすい
- **適切性**: スライドの内容と整合している

## トラブルシューティング

### スライドが正しく表示されない

**原因**: marp-syntax.mdに従っていない可能性があります。
**解決策**: フロントマター、スライド区切り、レイアウト指定を確認してください。

### デザインが統一されていない

**原因**: visual-hierarchy.mdに従っていない可能性があります。
**解決策**: フォントサイズ、カラースキーム、スペーシングを確認してください。

### 絵文字が多すぎる/少なすぎる

**原因**: emoji-usage.mdのガイドラインに従っていない可能性があります。
**解決策**: 1スライド1-2個の原則を守り、適切なカテゴリから選択してください。
