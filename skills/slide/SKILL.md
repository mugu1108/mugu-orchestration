# Slideスキル: Marpスライド生成

このスキルは、**Marp形式のプレゼンテーションスライドを自動生成**します。セミナー資料、ハンズオンワークショップ、製品紹介など、さまざまな用途に対応したテンプレートを提供し、ビジュアル階層とデザインの一貫性を保証します。

> **重要**: 「AIに丸投げ」ではなく「人間が設計した仕組みの中でAIが動く」アプローチを採用。
> スタイルガイドとレイアウトパターンを厳密に参照してスライドを生成します。

## 概要

Slideスキルは、以下の機能を提供します：

1. **テンプレート選択**: 用途に応じた3種類のテンプレート
   - **general-presentation**: 一般的な登壇・セミナー用
   - **hands-on-workshop**: ハンズオン形式ワークショップ用
   - **product-intro**: 製品・サービス紹介用

2. **デザインシステム**: 統一されたスタイルガイドに準拠
   - **カラーパレット**: グレースケール基調 + ネイビー#1B4565 + ティール#3E9BA4
   - **タイポグラフィ**: text-em-3xl〜text-em-base の5段階
   - **レイアウトパターン**: 40種類を7カテゴリで体系化
   - **Tailwind CSS統合**: グリッドレイアウト、パネルデザイン

3. **禁止事項の厳守**: プロフェッショナルなデザインのため
   - タイトルにコロンを使わない
   - 感嘆符・疑問符を避ける
   - 装飾的な絵文字を使わない
   - 派手な色（red-600, green-600等）を使わない

4. **自動コンテンツ生成**: トピックとターゲットに基づく
   - スライド構成の自動決定
   - レイアウトパターンからの選択
   - Marp構文の正確な適用

## 起動タイミング

以下の場合にこのスキルを使用してください：

- 技術カンファレンスやセミナーの登壇資料を作成したい
- ハンズオン形式のワークショップ資料が必要
- 製品やサービスの紹介スライドを作成したい
- Marp形式でプレゼンテーション資料を自動生成したい
- ビジュアル階層とデザインの一貫性を保ちたい

## 前提条件

- `skills/slide/templates/` に3種類のテンプレートが存在する
- `skills/slide/references/` にリファレンスドキュメントが存在する
  - **`style-guide.md`**: スタイルガイド（カラーパレット、禁止事項）【必読】
  - **`layout-patterns.md`**: 40種類のレイアウトパターン【必読】
  - `visual-hierarchy.md`: ビジュアル階層ガイドライン
  - `emoji-usage.md`: 絵文字使用ガイド
  - `marp-syntax.md`: Marp構文リファレンス
- Marp CLI（オプション、プレビュー用）
- Tailwind CSS（推奨、グリッドレイアウト用）

## ワークフローステップ

### ステップ1: 要件の収集

ユーザーに以下の情報を確認します：

1. **テンプレートタイプ**:
   - 一般登壇・セミナー（general-presentation）
   - ハンズオンワークショップ（hands-on-workshop）
   - 製品紹介（product-intro）

2. **対象者**: 初心者、中級者、上級者、その他

3. **時間**: プレゼンテーションの想定時間（例: 30分、1時間、2時間）

4. **トピック**: スライドの主題（例: "TypeScript入門"、"Next.js最新機能"）

5. **学習目標**: ユーザーに何を学んでほしいか（ワークショップの場合）

6. **出力ファイルパス**: スライドを保存する場所（デフォルト: `./slides/`）

必要に応じてAskUserQuestionツールを使用して情報を収集します。

### ステップ2: テンプレート選択

収集した要件に基づいて、最適なテンプレートを選択します：

```
ユーザーのリクエスト分析
    ↓
general-presentation.md: 技術カンファレンスやセミナー登壇
hands-on-workshop.md: 参加者が実際に手を動かす形式
product-intro.md: 自社製品やサービスの紹介
```

Readツールを使用して選択したテンプレートを読み込みます。

**テンプレート選択基準**:

- **キーワード "ハンズオン", "実習", "ワークショップ"** → hands-on-workshop.md
- **キーワード "製品", "サービス", "ツール紹介"** → product-intro.md
- **それ以外の登壇、セミナー、カンファレンス** → general-presentation.md

### ステップ3: アウトライン作成

トピックとターゲットに基づいて、スライドの構成を決定します。

**一般登壇・セミナー（general-presentation）の構成例**:
1. タイトルスライド
2. 自己紹介
3. アジェンダ
4. 問題提起
5. 解決策の提示
6. 技術詳細
7. 事例紹介
8. まとめとNext Steps

**ハンズオンワークショップ（hands-on-workshop）の構成例**:
1. タイトルスライド
2. 学習目標
3. アジェンダ
4. 前提知識の確認
5. セクション1: 基礎（実習含む）
6. セクション2: 応用（実習含む）
7. チャレンジ課題
8. まとめと次のステップ

**製品紹介（product-intro）の構成例**:
1. タイトルスライド
2. 解決する課題
3. ソリューション概要
4. 主要機能
5. デモ
6. ユースケース
7. 導入ステップ
8. まとめとCTA

### ステップ4: Marpマークダウン生成

選択したテンプレートをベースに、実際のスライドコンテンツを生成します。

**⚠️ 重要: 禁止事項（必須）**

以下のルールは**絶対に守ってください**：

**テキストの禁止事項**:
- ❌ タイトルにコロンを使わない（「原則: 自律性」→「原則 自律性」）
- ❌ 感嘆符を使わない（「素晴らしい!」→「素晴らしい」）
- ❌ 疑問符をスライド本文に使わない（Q&Aスライドは除く）
- ❌ 装飾的な絵文字を使わない（✅❌⚠️のみ許可）

**カラーの禁止事項**:
- ❌ red-600, green-600, blue-600などの派手な色を使わない
- ❌ 1スライドに3色以上のアクセントカラーを使わない
- ✅ グレースケール + ネイビー#1B4565 + ティール#3E9BA4 のみ使用

**⚠️ 重要: スライドはみ出し防止ルール（必須）**

スライドのコンテンツがはみ出すことは**絶対に許容されません**。以下のルールを厳守してください：

**コードブロックの制限**:
- コードブロックは**最大8行まで**（9行以上は分割必須）
- 1スライドに複数のコードブロックを含める場合は、**合計8行以内**
- 10行以上のコードは必ず複数スライドに分割

**⚠️ 重要: レイアウトパターンの使用（必須）**

毎回新しいHTMLを生成するのではなく、`layout-patterns.md`から適切なパターンを選択して使用してください。

**1スライドあたりのコンテンツ上限**:
- コードブロックのみ: 最大8行
- コードブロック + 説明テキスト: コード6行 + テキスト2-3行
- テキストのみ: 箇条書き最大6項目、または段落2-3個
- テーブル: 最大5行 × 4列

**分割の指針**:
```
❌ 悪い例: 1スライドに15行のコード + 説明3項目
✅ 良い例:
  - スライド1: コード前半7行 + 説明
  - スライド2: コード後半8行 + 説明
```

**スライド分割時の命名規則**:
- 「実装の詳細（1/3）」「実装の詳細（2/3）」「実装の詳細（3/3）」
- 「デモ: ステップ1」「デモ: ステップ2」

**フロントマター**:
```yaml
---
marp: true
theme: default
paginate: true
header: 'セミナータイトル'
footer: '© 2026 あなたの名前'
style: |
  section {
    font-family: 'Hiragino Sans', 'Noto Sans JP', sans-serif;
    background-color: #ffffff;
  }
  h1 {
    font-size: 48px;
    color: #1a202c;
    font-weight: bold;
  }
  h2 {
    font-size: 36px;
    color: #2d3748;
    font-weight: bold;
  }
  h3 {
    font-size: 28px;
    color: #4a5568;
  }
  p, li {
    font-size: 24px;
    line-height: 1.6;
  }
  code {
    background-color: #f7fafc;
    padding: 2px 6px;
    border-radius: 4px;
  }
  pre {
    background-color: #2d3748;
    color: #e2e8f0;
    padding: 16px;
    border-radius: 8px;
  }
---
```

**コンテンツスライド**:
- トピックに関連する具体的な内容を記述
- コード例、図表、リストを適切に配置
- 各スライドに1-2個の絵文字を追加（emoji-usage.mdに準拠）

### ステップ5: リファレンス適用

生成したスライドに、リファレンスガイドラインを適用します。

**visual-hierarchy.md の適用**:
- フォントサイズ階層の遵守（h1:48px, h2:36px, h3:28px, body:24px）
- カラースキームの一貫性（プライマリ、セカンダリ、アクセント）
- スペーシングの適切な使用（マージン、パディング）
- **コードブロックの配色**: ダーク背景に白（#ffffff）を基調とし、強調色は明るく鮮やかな色（Amber #fbbf24、Light Purple #a78bfa）を使用

**emoji-usage.md の適用**:
- カテゴリ別絵文字の選択（学習、技術、情報、状態など）
- 1スライド1-2個までの制限
- 文脈に適した絵文字の使用

**marp-syntax.md の適用**:
- スライド区切り（`---`）の正確な配置
- レイアウト指定（`<!-- _class: lead -->`など）
- 画像挿入、リスト、コードブロックの正確な構文

### ステップ6: 整合性チェック

生成したスライド全体の整合性を検証します。

**チェック項目**:
- [ ] フロントマターが正しく設定されている
- [ ] すべてのスライドがビジュアル階層に準拠している
- [ ] 絵文字が適切に使用されている（過度な使用がない）
- [ ] スライド区切り（`---`）が正しく配置されている
- [ ] コードブロックが正しくフォーマットされている
- [ ] **コードブロックの配色が明るく見やすい（白基調、明るい強調色）**
- [ ] 画像パスが有効である（画像を使用する場合）
- [ ] ヘッダーとフッターが設定されている
- [ ] **⚠️ コードブロックが8行以内である（必須）**
- [ ] **⚠️ 1スライドのコンテンツ量が上限を超えていない（必須）**
- [ ] **⚠️ 長いコードが適切に複数スライドに分割されている（必須）**

### ステップ7: ファイル保存

生成したMarpマークダウンを指定されたパスに保存します。

```
デフォルトパス: ./slides/{topic}-{date}.md
例: ./slides/typescript-intro-20260127.md
```

Writeツールを使用してファイルを保存します。

**レポート内容**:

## ベストプラクティス

### ⚠️ はみ出し防止（最重要）

**スライドのコンテンツがはみ出すことは絶対に許容されません。**

1. **コードブロックは8行以内**: 9行以上は必ず分割
2. **コード + テキストは控えめに**: コード6行 + テキスト2-3行が上限
3. **長いコードは論理単位で分割**:
   - 設定部分と実装部分を分ける
   - 関数ごとに分ける
   - インポート、定義、使用で分ける
4. **分割時はタイトルを連番に**: 「（1/3）」「（2/3）」「（3/3）」

**分割例**:
```
元のコード（20行）を以下のように分割：
- スライド1「実装（1/3）」: インポートと型定義（6行）
- スライド2「実装（2/3）」: メイン関数（8行）
- スライド3「実装（3/3）」: 使用例（6行）
```

### スライドデザイン

1. **ビジュアル階層を維持する**: visual-hierarchy.md に準拠し、フォントサイズとカラーを統一
2. **1スライド1メッセージ**: 各スライドには1つの主要なメッセージのみ
3. **適度な空白**: テキストを詰め込みすぎない、スペーシングを活用
4. **絵文字の適切な使用**: emoji-usage.md に従い、1スライド1-2個まで

### コンテンツ

1. **対象者を意識する**: 初心者向けは詳細に、上級者向けは要点のみ
2. **ストーリーを作る**: 問題 → 解決策 → 実例 の流れを意識
3. **コード例は8行以内**: 重要な部分のみを抽出、9行以上は必ず分割
4. **実習時間を確保**: ハンズオンの場合、実習時間を明示
5. **テキスト量も制限**: 箇条書きは6項目以内、段落は2-3個以内

### 技術的なポイント

1. **Marp構文の正確性**: marp-syntax.md を参照し、正しい構文を使用
2. **テンプレートのカスタマイズ**: 必要に応じてスタイルを調整
3. **ファイル命名規則**: `{topic}-{date}.md` 形式で保存
4. **バージョン管理**: 重要なスライドはGitで管理
5. **コードブロックの視認性**: ダーク背景には白基調（#ffffff）と明るい強調色（Amber #fbbf24）を使用し、はっきり見えるようにする

### テンプレート選択

1. **明確な基準**: キーワードベースで適切なテンプレートを選択
2. **ハイブリッド型も可能**: 複数のテンプレートを組み合わせることも検討
3. **カスタマイズの余地**: テンプレートは基礎、必要に応じて変更

## モデル選択

このスキルは、以下のモデルで最適に動作します：

- **Sonnet**: コンテンツ生成とデザイン判断のバランスが取れたモデル（推奨）
- **Opus**: 高度なコンテンツ生成が必要な場合（複雑なトピック、専門的な内容）
- **Haiku**: シンプルなスライド生成や既存スライドの修正（コスト重視）
