# Slideスキル テンプレート一覧

このディレクトリには、Marp形式のプレゼンテーションスライドを生成するための3種類のテンプレートが含まれています。

## テンプレート一覧

### 1. general-presentation.md
**用途**: 一般的な登壇・セミナー用

技術カンファレンス、セミナー登壇、勉強会など、標準的なプレゼンテーション形式に適したテンプレートです。

**主要セクション**:
- タイトルスライド
- 自己紹介
- アジェンダ
- 問題提起
- 解決策の提示
- 技術詳細
- 事例紹介
- まとめとNext Steps

**推奨時間**: 20-60分

**対象者**: 初心者〜上級者

**使用例**:
- "Next.js 15の新機能についてセミナー資料を作成"
- "TypeScriptのベストプラクティスについて登壇資料を作成"
- "React Server Componentsの解説スライドを作成"

---

### 2. hands-on-workshop.md
**用途**: ハンズオン形式ワークショップ用

参加者が実際に手を動かしながら学ぶ形式のワークショップに適したテンプレートです。実習セクションとステップバイステップの説明が充実しています。

**主要セクション**:
- タイトルスライド
- 学習目標
- アジェンダ
- 前提知識の確認
- 環境構築（実習）
- セクション1: 基礎（実習含む）
- セクション2: 応用（実習含む）
- セクション3: 統合（実習含む）
- チャレンジ課題
- まとめと次のステップ

**推奨時間**: 1-4時間

**対象者**: 初心者〜中級者

**使用例**:
- "TypeScript入門のワークショップ資料を作成"
- "Supabase実践ハンズオンの資料を作成"
- "Git/GitHub実習ワークショップのスライドを作成"

---

### 3. product-intro.md
**用途**: 製品・サービス紹介用

自社製品、オープンソースツール、サービスの紹介に適したテンプレートです。課題→ソリューション→機能→デモの流れで構成されています。

**主要セクション**:
- タイトルスライド
- 解決する課題
- ソリューション概要
- 主要機能
- デモ（ライブまたはスクリーンショット）
- ユースケース
- 導入ステップ
- まとめとCTA（Call To Action）

**推奨時間**: 15-30分

**対象者**: 全レベル

**使用例**:
- "Supabaseの製品紹介スライドを作成"
- "社内ツールの紹介資料を作成"
- "OSSプロジェクトの紹介スライドを作成"

## テンプレート選択ガイド

### どのテンプレートを選ぶべきか？

以下のフローチャートに従ってテンプレートを選択してください：

```
参加者が実際に手を動かす？
├─ はい → hands-on-workshop.md
└─ いいえ → 製品・サービスを紹介する？
           ├─ はい → product-intro.md
           └─ いいえ → general-presentation.md
```

### キーワードベースの選択

リクエストに以下のキーワードが含まれている場合：

- **hands-on-workshop.md**:
  - "ハンズオン", "実習", "ワークショップ", "workshop", "hands-on"
  - "実際に手を動かす", "環境構築", "ステップバイステップ"

- **product-intro.md**:
  - "製品", "サービス", "ツール紹介", "product", "service", "tool"
  - "デモ", "機能紹介", "導入方法"

- **general-presentation.md**:
  - "セミナー", "登壇", "カンファレンス", "勉強会"
  - "解説", "紹介", "ベストプラクティス", "新機能"

## カスタマイズ方法

### スタイルのカスタマイズ

各テンプレートのフロントマターにある`style:`セクションを編集することで、デザインをカスタマイズできます。

```yaml
style: |
  section {
    background-color: #ffffff;  # 背景色を変更
  }
  h1 {
    font-size: 48px;            # 見出しサイズを変更
    color: #1a202c;             # 見出し色を変更
  }
```

### セクションの追加・削除

テンプレートはあくまで基礎です。必要に応じてセクションを追加・削除してください。

**追加の例**:
- Q&Aセクション
- 参考資料
- 付録
- コントリビューター紹介

**削除の例**:
- 時間が短い場合は詳細セクションを削減
- 対象者に合わせて基礎セクションをスキップ

### ビジュアル階層の遵守

カスタマイズする際も、`references/visual-hierarchy.md`に記載されたデザインガイドラインを遵守してください：

- フォントサイズ階層: h1(48px) > h2(36px) > h3(28px) > body(24px)
- カラースキーム: プライマリ、セカンダリ、アクセント
- スペーシング: 適切なマージンとパディング

### 絵文字の使用

`references/emoji-usage.md`を参照して、適切な絵文字を選択してください。

**ルール**:
- 1スライド1-2個まで
- カテゴリ別に整理された絵文字リストから選択
- 過度な使用は避ける

## テンプレートの構造

すべてのテンプレートは以下の構造に従っています：

```markdown
---
marp: true
theme: default
paginate: true
header: 'セミナータイトル'
footer: '© 2026 あなたの名前'
style: |
  # スタイル定義
---

# タイトルスライド

---

# セクション1
<!-- セクション内容 -->

---

# セクション2
<!-- セクション内容 -->

...
```

## ベストプラクティス

1. **テンプレートはベース**: テンプレートを出発点として、ニーズに合わせて調整
2. **一貫性を保つ**: スタイルやトーンの一貫性を維持
3. **対象者を意識**: 初心者向けは詳細に、上級者向けは要点のみ
4. **時間配分**: スライド数と発表時間のバランスを考慮（1スライド約1-2分）
5. **実習時間**: ハンズオンの場合、実習時間を明示的に記載
