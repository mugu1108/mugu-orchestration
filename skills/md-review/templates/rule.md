# .claude/rules/ テンプレート

## フロントマター

```yaml
---
paths:
  - "src/**/*.ts"        # スコープ限定の場合
---
```

- `paths` あり → 該当パスのファイル操作時のみ適用
- `paths` なし → 全セッションで常時適用
- フロントマターなしも可（常時適用と同義）

## 構造

```markdown
# タイトル（短く、何のルールか一目でわかる）

簡潔な導入文（1行、任意）

## カテゴリ名（必要な場合のみ）

- 具体的で行動可能なルール
- 「〜する」「〜しない」の形式
- 1項目1行、説明が必要なら次行にインデント
```

## 必須要素

- H1タイトル
- 箇条書きのルール（最低1つ）

## ガイドライン

- 5〜20行程度が理想。短いほど良い
- 各ルールは「この行がなかったらClaudeがミスするか？」で取捨選択
- 自明なルール（「バグを書くな」）は不要
- 詳細なルールは `rules/` 配下に分離し、`.claude/rules/` は要点のみ
- コードブロックは最小限（必要な場合のみ）
