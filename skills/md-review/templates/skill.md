# skills/*/SKILL.md テンプレート

## フロントマター（必須）

```yaml
---
name: skill-name              # ケバブケース
description: 1〜2文の説明
category: analysis             # analysis / workflow / productivity / documentation
triggers:
  - "英語トリガー"
  - "日本語トリガー"
tools:
  - Read
  - Glob
  - Task
---
```

## 構造

```markdown
# /コマンド名 - 短い説明

**説明**: 1文でスキルの目的を述べる。

## 使用方法

（コードブロックで構文を示す）

## 概要

機能の概要を2〜3文で。

## 起動タイミング

- いつこのスキルを使うべきか（箇条書き）

## 前提条件

- 必要な設定・環境・ファイル

## ワークフローステップ

### ステップ1: タイトル
内容

### ステップ2: タイトル
内容

## 使用例

### 例1: タイトル
具体的な入力と期待される出力

## ベストプラクティス

1. 番号付きリスト

## 関連エージェント

- **エージェント名**: 役割（パスへのリンク）

## モデル選択

- **Sonnet**: 通常用途
- **Opus**: 高精度が必要な場合

## 注意事項

- 制約や注意点
```

## 必須セクション

1. フロントマター（5キー全て）
2. H1タイトル（`/コマンド名 - 説明` 形式）
3. 使用方法
4. 概要
5. ワークフローステップ
6. 使用例（最低1つ）

## ガイドライン

- ワークフローは番号付きステップで、各ステップにH3見出し
- 使用例はコードブロックで入力と出力を明示
- 出力フォーマットがある場合はMarkdownコードブロックで定義
- `triggers` は英語・日本語の両方を含める
