# commands/*.md テンプレート

## フロントマター

なし（commandsファイルにフロントマターは不要）

## 構造

```markdown
# /コマンド名 - 短い説明

**説明**: 1文でコマンドの目的を述べる。

## 使用方法

（コードブロックで構文とオプションを示す）

## 概要

コマンドの機能を2〜3文で。

## いつ使うか

- 使用すべき場面（箇条書き）

## 使用例

### 例1: タイトル

入力:
（コードブロック）

出力:
（コードブロック）

## オプション（ある場合）

| オプション | 説明 |
|-----------|------|
| `--flag` | 効果 |

## ベストプラクティス

1. 番号付きリスト

## 関連エージェント

- **エージェント名**: 役割（`agents/xxx.md`）

## モデル選択

- **Sonnet**: 通常用途
- **Opus**: 高精度が必要な場合

## 詳細情報

- 参照先へのリンク

## 注意事項

- 制約や注意点
```

## 必須セクション

1. H1タイトル（`/コマンド名 - 説明` 形式）
2. 使用方法
3. 概要
4. いつ使うか
5. 使用例（最低1つ）

## SKILL.md との役割分担

commands はユーザー向けクイックリファレンス、SKILL.md はClaudeの動作定義:

| 書く場所 | 内容 |
|---------|------|
| **commands/** | 使用方法、いつ使うか、使用例（入力+出力1例）、オプション |
| **SKILL.md** | ワークフロー詳細、出力フォーマット定義、エージェント呼び出し方法 |

- 重複を避ける。ワークフローや出力フォーマットの詳細は `SKILL.md を参照` で済ませる
- ベストプラクティス・注意事項も SKILL.md に1箇所で定義し、commands では繰り返さない

## ガイドライン

- 使用例は1〜2例に絞る。出力例は構造がわかる程度に（全文はSKILL.mdに委ねる）
- `## いつ使うか` = SKILL.md の `## 起動タイミング` と対応（文言を合わせる、コピーしない）
- `## オプション` は実装済みのもののみ記載。未実装オプションは書かない
- commands 単体で「何ができるか」がわかり、「どう動くか」はSKILL.mdに誘導する構成にする
