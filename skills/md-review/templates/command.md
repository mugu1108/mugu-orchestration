# commands/*.md テンプレート

## フロントマター

なし（commandsファイルにフロントマターは不要）

## 構造

```markdown
# /コマンド名 - 短い説明

**説明**: 1文でコマンドの目的を述べる。

## 使用方法

（コードブロックで構文とオプションを示す）

## 概要

コマンドの機能を2〜3文で。

## いつ使うか

- 使用すべき場面（箇条書き）

## 使用例

### 例1: タイトル

入力:
（コードブロック）

出力:
（コードブロック）

## オプション（ある場合）

| オプション | 説明 |
|-----------|------|
| `--flag` | 効果 |

## ベストプラクティス

1. 番号付きリスト

## 関連エージェント

- **エージェント名**: 役割（`agents/xxx.md`）

## モデル選択

- **Sonnet**: 通常用途
- **Opus**: 高精度が必要な場合

## 詳細情報

- 参照先へのリンク

## 注意事項

- 制約や注意点
```

## 必須セクション

1. H1タイトル（`/コマンド名 - 説明` 形式）
2. 使用方法
3. 概要
4. いつ使うか
5. 使用例（最低1つ）

## ガイドライン

- `commands/*.md` は `skills/*/SKILL.md` と対になることが多い
- 使用例は最も詳細に書く（入力と期待される出力の両方）
- `## いつ使うか` = SKILL.md の `## 起動タイミング` と対応
- 出力例はMarkdownコードブロックで完全に示す
