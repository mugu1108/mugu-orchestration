# MDファイル品質ルール

MDファイル（rules、skills、agents、commands）を書く・編集するとき、以下を守る。

## テンプレート参照（必須）

MDファイルを新規作成・大幅編集する前に、該当テンプレートを **必ず Read で読んでから** 書き始める:

| 対象パス | テンプレート |
|---------|-------------|
| `.claude/rules/*.md` | `skills/md-review/templates/rule.md` |
| `skills/*/SKILL.md` | `skills/md-review/templates/skill.md` |
| `agents/**/*.md` | `skills/md-review/templates/agent.md` |
| `commands/*.md` | `skills/md-review/templates/command.md` |

## 構造

- YAMLフロントマター → 概要 → 詳細の順で書く
- テンプレートの必須セクションを満たしてから任意セクションを追加
- 1ファイル500行以下。超えたら分割を検討

## 簡潔さ

- 「この行を消したらClaudeがミスするか？」で各行の必要性を判断
- 自明なルール（「バグを書くな」等）は書かない
- 同じことを別の言い回しで繰り返さない

## 具体性

- 「きれいに書け」ではなく「2スペースインデント」のように具体的に
- 悪い例/良い例のペアで示す（非自明な場合のみ）
- 行動可能な指示にする。感想や願望は書かない

## アンチパターン

- 冗長な背景説明（Claudeは文脈を理解している）
- 重複するルール（1箇所で定義、他は参照）
- フロントマターの形式間違い（種類ごとに異なる）
