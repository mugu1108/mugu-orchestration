## コンテンツ作成・ドキュメント系

### 1. blog-writer - ブログ記事生成スキル

**目的**: 技術ブログ、Zenn、Qiita向けの記事を自動生成

#### 使用例

```
"Next.js 15のApp Routerについてブログ記事を書いて"
```

#### 生成物

- SEO最適化されたタイトル（5-10パターン）
- 目次構成
- コード例付き本文
- Zenn/Qiita形式のマークダウン
- OGP画像の提案
- ハッシュタグ提案

#### ワークフロー

```
1. トピックの分析
   - 対象読者の特定
   - 記事の目的明確化
   - SEOキーワード抽出

2. アウトライン作成
   - 導入（問題提起）
   - 本文（解決策・説明）
   - まとめ（Next Steps）

3. コンテンツ生成
   - 各セクションの執筆
   - コード例の追加
   - 図表の提案

4. 最適化
   - SEOチェック
   - 可読性チェック
   - リンク追加

5. 保存
   - Zenn/Qiita形式のマークダウン
   - Notionに下書き保存
```

#### 実装のポイント

```javascript
// skills/blog-writer/SKILL.md

const blogOutline = {
  title: "Next.js 15のApp Routerで始めるモダン開発",
  targetAudience: "中級者",
  seoKeywords: ["Next.js 15", "App Router", "React Server Components"],
  sections: [
    {
      title: "はじめに",
      content: "App Routerが解決する問題...",
      codeExamples: []
    },
    {
      title: "App Routerの基本",
      content: "ディレクトリ構造と命名規則...",
      codeExamples: [
        {
          language: "typescript",
          code: "// app/page.tsx\nexport default function Home() { ... }"
        }
      ]
    }
  ]
};
```

#### テンプレート

```markdown
---
title: "{{TITLE}}"
emoji: "{{EMOJI}}"
type: "tech"
topics: [{{TOPICS}}]
published: false
---

# {{TITLE}}

## はじめに

{{INTRO}}

## {{SECTION_1_TITLE}}

{{SECTION_1_CONTENT}}

\`\`\`{{LANGUAGE}}
{{CODE_EXAMPLE_1}}
\`\`\`

## まとめ

{{SUMMARY}}

## 参考資料

- [{{REFERENCE_1}}]({{URL_1}})
- [{{REFERENCE_2}}]({{URL_2}})
```

### 2. meeting-notes - 議事録生成スキル

**目的**: MTG音声・テキストから議事録を自動生成

#### 使用例

```
"今日のクライアントMTGの音声を議事録にして"
```

#### 生成物

- 議題・参加者
- 決定事項
- アクションアイテム（担当者・期限）
- 次回MTGの議題
- Notion/Slackに自動投稿

#### ワークフローン

```
1. 音声/テキスト入力
   - 音声ファイルアップロード
   - またはテキスト入力

2. 文字起こし（音声の場合）
   - Whisper APIまたはGoogle Speech-to-Text

3. 内容分析
   - 重要な発言の抽出
   - 決定事項の特定
   - アクションアイテムの抽出

4. 議事録生成
   - フォーマット化
   - 担当者・期限の明記

5. 保存・共有
   - Notionに保存
   - Slackに投稿
   - 関係者にメンション
```

#### テンプレート

```markdown
# {{MEETING_TITLE}} 議事録

**日時**: {{DATE}} {{TIME}}
**参加者**: {{ATTENDEES}}
**場所**: {{LOCATION}}

## 議題

1. {{AGENDA_1}}
2. {{AGENDA_2}}
3. {{AGENDA_3}}

## 決定事項

1. **{{DECISION_1}}**
   - 理由: {{REASON_1}}

2. **{{DECISION_2}}**
   - 理由: {{REASON_2}}

## アクションアイテム

- [ ] {{ACTION_1}} (@{{ASSIGNEE_1}} 期限: {{DUE_DATE_1}})
- [ ] {{ACTION_2}} (@{{ASSIGNEE_2}} 期限: {{DUE_DATE_2}})
- [ ] {{ACTION_3}} (@{{ASSIGNEE_3}} 期限: {{DUE_DATE_3}})

## 次回MTGの議題

1. {{NEXT_AGENDA_1}}
2. {{NEXT_AGENDA_2}}

## 補足

{{NOTES}}
```

## 情報整理・知識管理系

### 4. knowledge-organizer - 情報整理スキル

**目的**: 散らばった情報をNotion DBに整理

#### 使用例

```
"今週保存したブックマークをカテゴリ分けしてNotionに保存"
```

#### 機能

- URL先の内容を要約
- 自動タグ付け
- 重要度スコアリング
- 関連記事の紐付け
- "あとで読む"リストの管理

#### ワークフロー

```
1. 情報源の収集
   - ブラウザブックマーク
   - Slackで共有されたリンク
   - Pocketの記事

2. コンテンツ分析
   - タイトル・本文の抽出
   - キーワード分析
   - カテゴリ自動分類

3. 要約生成
   - 3行要約
   - 重要ポイント抽出

4. Notion保存
   - カテゴリDB振り分け
   - タグ付け
   - 重要度設定

5. 関連付け
   - 既存記事との関連性分析
   - リンク作成
```

#### Notion DB構造

```
【ナレッジベースDB】

プロパティ:
- タイトル (Title)
- URL (URL)
- カテゴリ (Select): 技術/ビジネス/デザイン/その他
- タグ (Multi-select)
- 重要度 (Select): 高/中/低
- ステータス (Select): 未読/読了/要再読
- 要約 (Text)
- 保存日 (Date)
- 最終閲覧 (Date)
```

## プロジェクト管理・タスク管理系

### 9. task-prioritizer - タスク優先順位付けスキル

**目的**: タスクを自動で優先順位付け

#### 使用例

```
"今週のタスクを優先順位付けして"
```

#### 分析項目

- 緊急度（期限）
- 重要度（ビジネスインパクト）
- 依存関係
- 工数見積もり
- 推奨スケジュール

#### 生成物

- 優先順位付きタスクリスト
- 1日のタスクプラン
- 週間計画
- Notionに自動反映

#### ワークフロー

```
1. タスク収集
   - NotionのタスクDB
   - カレンダーの予定
   - Slackのリマインダー

2. スコアリング
   - 緊急度スコア（期限まで の日数）
   - 重要度スコア（ビジネスインパクト）
   - 工数見積もり
   - 総合スコア = 緊急度 × 重要度 / 工数

3. 優先順位付け
   - アイゼンハワーマトリクスに配置
   - 依存関係を考慮した順序決定

4. スケジュール生成
   - 1日のキャパシティ（8時間）を考慮
   - 実現可能なプラン作成

5. Notion更新
   - 優先順位を自動設定
   - 推奨実施日を設定
```

#### 使用例

```
"レシート画像から経費を記録"
```

- OCRでレシート読み取り
- 自動カテゴリ分類

```
4. Notion保存
   - 経費管理DB
   - ステータス: 未確定/確定

5. Slack通知
   - 記録完了通知
   - 確認・修正ボタン

6. 月次集計
   - カテゴリ別集計
   - グラフ生成
   - 確定申告用CSV出力
```

```
【経費管理DB】

プロパティ:
- 店舗名 (Title)
- 金額 (Number)
- 日付 (Date)
- カテゴリ (Select): 交通費/食費/通信費/書籍/その他
- 勘定科目 (Select): 旅費交通費/会議費/通信費/新聞図書費
```

### 14. focus-session - 集中セッション管理スキル

**目的**: ポモドーロタイマー + 自動記録

#### 使用例

```
"/focus start スライド作成"
```

- 25分タイマー開始

- 作業開始をSupabaseに記録

#### ワークフロー

```
1. セッション開始
   - タスク名を記録

3. 記録
   - Supabaseに開始時刻記録

4. 25分後
   - タイマー終了通知
   - 休憩開始（5分）
   - Supabaseに終了時刻記録

5. 統計
   - 1日の集中時間集計
   - 週次レポート生成
   - Notionにグラフ表示
```

### スコアリング基準

各スキルを以下の基準で評価（1-5点）：

1. **頻度**: どのくらい使うか
   - 5点: 毎日
   - 4点: 週3-5回
   - 3点: 週1-2回
   - 2点: 月数回
   - 1点: 稀

2. **時短効果**: 1回あたりの時短効果
   - 5点: 2時間以上
   - 4点: 1-2時間
   - 3点: 30分-1時間
   - 2点: 15-30分
   - 1点: 15分未満

3. **実装難易度**（逆スコア）
   - 5点: 非常に簡単（1日）
   - 4点: 簡単（2-3日）
   - 3点: 中程度（1週間）
   - 2点: 難しい（2週間）
   - 1点: 非常に難しい（1ヶ月以上）

4. **依存性**（他の機能への依存）
   - 5点: 依存なし（単独で動作）
   - 3点: 部分的依存
   - 1点: 高依存（他機能必須）

**総合スコア** = (頻度 + 時短効果) × 実装難易度 × 依存性
